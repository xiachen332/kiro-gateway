# Kiro Gateway 架构概览

## 系统定位

Kiro Gateway 是一个 LLM API 网关，提供统一的 API 接口，支持多提供商（OpenAI、MiniMax 等）的协议转换和路由。

## 架构模式

```
┌─────────────────────────────────────────────────────────┐
│                      客户端请求                          │
└─────────────────────┬───────────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────────┐
│                   FastAPI 路由层                         │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────────────┐│
│  │ /v1/chat/   │ │ /v1/models  │ │ /health             ││
│  │ completions │ │             │ │                     ││
│  └──────┬──────┘ └──────┬──────┘ └─────────────────────┘│
└─────────┼───────────────┼───────────────────────────────┘
          │               │
          ▼               ▼
┌─────────────────────────────────────────────────────────┐
│                   核心网关逻辑 (kiro/)                    │
│                                                          │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌─────────┐ │
│  │ 认证     │  │ 限流     │  │ 模型解析 │  │ 协议转换 │ │

│  └────┬─────┘  └────┬─────┘  └────┬─────┘  └────┬────┘ │

│                          │                               │

│  ┌──────────────────────────────────────────────────┐   │
│  │              上游提供商 (Upstream)                  │   │
│  │  ┌──────────┐ ┌──────────┐ ┌──────────────────┐  │   │

```

## 核心模块

| 模块 | 文件 | 职责 |
|------|------|------|
| **配置** | `kiro/config.py` | 环境变量、配置加载、默认值 |
| **认证** | `kiro/auth.py` | API Key 验证、权限检查 |
| **限流** | `kiro/rate_limit.py` | 请求频率控制、配额管理 |
| **模型解析** | `kiro/model_resolver.py` | 模型名称映射、提供商选择 |
| **协议转换** | `kiro/converters_*.py` | 请求/响应格式转换 |
| **数据模型** | `kiro/models_*.py` | Pydantic 模型定义 |
| **路由** | `kiro/routes_*.py` | API 端点定义、请求处理 |

## 数据流

```
1. 客户端发送请求 (OpenAI 格式)
        ↓
2. 路由层解析请求
        ↓
3. 认证模块验证 API Key
        ↓
4. 限流模块检查配额
        ↓
5. 模型解析模块确定上游提供商
        ↓
6. 协议转换模块转换请求格式
        ↓
7. 发送到上游提供商
        ↓
8. 接收响应并转换回 OpenAI 格式
        ↓
9. 返回给客户端
```

## 技术栈

- **Web 框架**: FastAPI
- **服务器**: Hypercorn (HTTP/2 + ASGI)
- **进程管理**: PM2
- **配置**: Pydantic Settings + 环境变量
- **日志**: 标准 logging (待迁移到 structlog)
- **测试**: pytest

## 扩展点

1. **新增提供商**: 添加 `converters_{provider}.py` + `routes_{provider}.py`
2. **新增认证方式**: 扩展 `kiro/auth.py`
3. **新增限流策略**: 扩展 `kiro/rate_limit.py`
4. **新增模型**: 修改配置文件或数据库

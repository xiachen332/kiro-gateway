# Skill: Bug Fix（Bug 修复）

## 触发条件
- 收到 bug 报告
- 测试失败
- 服务异常

## 执行流程

### Step 1: 理解问题
1. 阅读 bug 描述
2. 复现问题（如可能）
3. 查看相关代码
4. 查看日志（pm2 logs / docs/troubleshooting/）

### Step 2: 定位根因
1. 使用二分法缩小范围
2. 检查输入数据
3. 检查边界条件
4. 检查外部依赖

### Step 3: 实施修复
1. 编写修复代码
2. 添加/更新测试
3. 运行测试验证
4. 运行 linter

### Step 4: 验证
1. 本地启动服务
2. 测试修复是否有效
3. 检查是否引入新问题
4. 更新文档（如需要）

### Step 5: 提交
1. git add .
2. git commit -m "fix(module): description"
3. git push fork main
4. 部署（如需要）

## 检查清单
- [ ] 理解 bug 根因
- [ ] 修复最小化（只改必要代码）
- [ ] 添加回归测试
- [ ] 所有测试通过
- [ ] linter 通过
- [ ] 本地验证通过
- [ ] 文档更新（如接口变更）

# Skill: Refactor（重构）

## 触发条件
- 代码质量下降
- 技术债务积累
- 性能问题
- 重复代码

## 执行流程

### Step 1: 识别问题
1. 运行质量报告：`python tools/quality_report.py`
2. 运行结构检查：`python tools/linters/check_structure.py`
3. 查看代码重复
4. 查看性能瓶颈

### Step 2: 制定计划
1. 确定重构范围
2. 评估风险
3. 确保有测试覆盖
4. 制定回滚方案

### Step 3: 小步重构
1. 每次只改一个模块
2. 保持测试通过
3. 提交小粒度 commit

### Step 4: 验证
1. 所有测试通过
2. 功能无回归
3. 性能对比
4. 代码质量改善

## 黄金原则
- 重构前先确保测试覆盖
- 小步提交，不要大改
- 保持行为不变
- 重构后更新文档
